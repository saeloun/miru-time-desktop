# Miru Time Tracking 0.1.3

Miru Time Tracking 0.1.3 improves the desktop timer, live timesheet navigation, Miru web sync, and release verification. It adds a resumable timer stack for starting new work without losing the previous timer, plus detailed Today, Week, and History entry views from the summary cards.

## Downloads

- macOS arm64 ZIP: https://github.com/saeloun/miru-time-desktop/releases/download/v0.1.3/Miru.Time.Tracking-darwin-arm64-0.1.3.zip
- Linux x64 ZIP: https://github.com/saeloun/miru-time-desktop/releases/download/v0.1.3/Miru.Time.Tracking-linux-x64-0.1.3.zip
- Windows x64 ZIP: https://github.com/saeloun/miru-time-desktop/releases/download/v0.1.3/Miru.Time.Tracking-win32-x64-0.1.3.zip

## Highlights

- Added API-backed Electron coverage for project and time entry loading.
- Added detailed time entry drill-down from Today, This week, and Entries summary cards.
- Added a paused timer stack for start-new-timer and resume flows.
- Added multi-timer push/pull support for Miru current timer sync payloads.
- Added login, signup confirmation, and workspace switching coverage.
- Added current timer pull/push coverage for the desktop-to-Miru sync path.
- Added multiple-timer push and pull Electron coverage.
- Added save-running-timer coverage for the desktop timer entry flow.
- Added manual entry create, edit, and resume coverage.
- Isolated each Electron E2E test with a fresh user data directory.
- Hardened teardown so full runs do not leave rogue Miru/Electron processes.
- Stabilized account menu action tests by scoping clicks to the visible account dialog.
