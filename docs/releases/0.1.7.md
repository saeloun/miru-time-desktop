# Miru Time Tracking 0.1.7

Miru Time Tracking 0.1.7 fixes macOS distribution for public desktop downloads. The macOS release build now uses Saeloun's Developer ID signing identity, submits the app for Apple notarization, and verifies the stapled ticket before release.

## Downloads

- macOS Apple Silicon ZIP: https://github.com/saeloun/miru-time-desktop/releases/download/v0.1.7/Miru.Time.Tracking-darwin-arm64-0.1.7.zip
- macOS Intel ZIP: https://github.com/saeloun/miru-time-desktop/releases/download/v0.1.7/Miru.Time.Tracking-darwin-x64-0.1.7.zip
- Linux x64 ZIP: https://github.com/saeloun/miru-time-desktop/releases/download/v0.1.7/Miru.Time.Tracking-linux-x64-0.1.7.zip
- Windows x64 ZIP: https://github.com/saeloun/miru-time-desktop/releases/download/v0.1.7/Miru.Time.Tracking-win32-x64-0.1.7.zip

## Fixes

- Signs public macOS ZIP builds with the `Developer ID Application: Saeloun Inc (R9V7L93928)` identity.
- Notarizes macOS release builds through Apple's notary service before ZIP packaging.
- Uses the stable macOS bundle identifier `com.saeloun.miru-time-tracking`.
- Fails release packaging when signing or notarization credentials are missing instead of producing an unsigned public build.
- Documents `codesign`, Gatekeeper, and stapler checks for every macOS release.

## Verification

- `rtk mise exec -- bun run check`
- `rtk mise exec -- bun run test`
- `rtk mise exec -- bun run package`
- `rtk mise exec -- bun run test:e2e`
- `rtk mise exec -- bun run make:mac:release`
- `rtk mise exec -- bun run make:linux`
- `rtk mise exec -- bun run make:windows`
- `codesign --verify --deep --strict --verbose=2` for macOS Apple Silicon and Intel apps
- `spctl --assess --type execute --verbose=4` for macOS Apple Silicon and Intel apps
- `xcrun stapler validate` for macOS Apple Silicon and Intel apps
